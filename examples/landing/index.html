Quickstart

Up and running in four steps.

No build pipeline. Drop the plugin into AstrBot, fill in a master admin key, and issue tokens to your friends.

# 1. drop the plugin into AstrBot's plugin directory
git clone https://github.com/Naomi-Armitage/astrbot_plugin_webchat_gateway.git \
  data/plugins/astrbot_plugin_webchat_gateway

# 2. install the SQLite driver (default backend)
pip install aiosqlite

# optional — if you'd rather use MySQL in production
pip install aiomysql
Features

Built for sharing an LLM, not exposing it.

Every primitive you'd otherwise have to bolt on yourself, already there and audited.

Per-friend tokens

Independent SHA-256 hashed tokens, shown exactly once at issue time. Revoke a single friend without touching anyone else.

Daily quotas

Each token has its own daily request budget that resets at midnight, with live remaining-count headers for clients to show.

Single-flight

The same token may have only one in-flight LLM call at a time, so a runaway script can't fan out and drain your quota.

IP brute-force shield

Repeated auth failures from a single IP earn an automatic temporary block, with a Retry-After hint to clients.

Audit log

Issue, revoke, chat, quota-exhaustion and IP-block events all land in a queryable audit_log table.

SQLite or MySQL

Zero-config SQLite for solo hosting, or swap in MySQL for production via a single DSN string — same schema, same code.
